GA4 Implementation Case Study

從零到自動化分析：世新美食誌的 GA4 數據串接實作歷程

本篇文章完整記錄了如何將 Google Analytics 4 整合至前端網頁，追蹤 YouTube 影音點擊自訂事件，並運用 Python 串接 Google Analytics Data API 進行自動化報告產出的全過程。

📅 發表日期：2026-05-26 • 👤 作者：世新美食誌開發團隊 • GA4 實作分享

在建置「世新美食誌」網站後，我們希望能夠深入瞭解使用者在網站上的實際行為。例如：他們最常瀏覽哪一家推薦美食？有多少人對我們的「影音特別企劃」感興趣並點擊播放？為了解答這些問題，我們規劃並完成了 Google Analytics 4 (GA4) 的深度串接。

1 基本追蹤碼埋設 (Global Page Tag)

首先，我們在 Google Analytics 後台建立了專屬的 GA4 資源，取得衡量 ID G-HDB1KMXVL4。接著在網頁 <head> 中引入官方的 gtag.js 追蹤碼，確保網站上的基本行為（如：page_view 頁面瀏覽量、first_visit 首次造訪、session_start 工作階段開始）可以自動被收集。 

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-HDB1KMXVL4"></script>
<script>
    window.dataLayer = window.dataLayer || [];
    function gtag(){dataLayer.push(arguments);}
    gtag('js', new Date());
    gtag('config', 'G-HDB1KMXVL4');
</script>

2 自訂事件追蹤：偵測 YouTube 影片播放

除了一般點擊外，我們希望特別追蹤對「影音企劃」感興趣的使用者。我們利用了 YouTube Iframe Player API 來擷取影片播放器的狀態變化。當檢測到使用者點擊播放按鈕（狀態為 YT.PlayerState.PLAYING）時，立即向 GA4 發送一個名為 interested_user 的自訂事件： 

var trackedPlay = false; // 避免重複發送
function onPlayerStateChange(event) {
    if (event.data == YT.PlayerState.PLAYING && !trackedPlay) {
        if (typeof gtag === 'function') {
            gtag('event', 'interested_user', {
                'event_category': 'video',
                'event_label': 'YouTube Video Play'
            });
            console.log('GA4 Event sent: interested_user');
            trackedPlay = true; // 標記已發送，避免重複計算
        }
    }
}

💡 實作亮點：

透過設定 trackedPlay 旗標，系統可以避免使用者重複點擊暫停／播放時發送重複事件，確保數據的真實性。

3 自動化數據提取 (Google Analytics Data API)

為了不需要每次都登入 GA 後台看報表，我們使用 Python 串接了 Google Analytics Data API v1beta。 我們在本地環境中利用 Google 服務帳戶認證金鑰，撰寫自動化腳本來撈取每日的流量明細與自訂事件報表： 

from google.analytics.data_v1beta import BetaAnalyticsDataClient
from google.analytics.data_v1beta.types import RunReportRequest, DateRange, Dimension, Metric

client = BetaAnalyticsDataClient()
request = RunReportRequest(
    property="properties/538936221", # 世新美食誌 GA4 資源 ID
    dimensions=[Dimension(name="eventName")],
    metrics=[Metric(name="eventCount"), Metric(name="activeUsers")],
    date_ranges=[DateRange(start_date="2026-05-01", end_date="today")]
)
response = client.run_report(request=request)
